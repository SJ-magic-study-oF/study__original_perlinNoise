                                                  Flafla2 Mod EleventhWorld 3 years ago

                                                  Hi, let me try and clear up some of your confusion. Firstly, the technical aspect of this article is meant to serve as an explanation of how the algorithm itself works. The permutation table, for example, is just an implementation detail.

                                                  As for how to use the algorithm: one way to think of it is of a 3D image with infinite resolution. The input is a 3D "coordinate" in the image. Check out figure 4 - it is a representation of 2D perlin noise. The lines are the unit squares (each integral coordinate). So you want to use decimal numbers as input coordinates. One more thing to keep in mind is that if you only use integers (no decimals) then you will only get 0.5 as your output. See my comment with @g c for more info.

                                                  In this context, what really matters is the scale that you sample the perlin noise at. Larger scales mean larger vaiations in terrain, clouds, or whatever you are using the noise for, and smaller scales mean smaller variations. See the section on Octaves for more info regarding this.
